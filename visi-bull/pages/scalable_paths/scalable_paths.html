Scalable SVG Paths

-- --, 2014 {% endblock %} {% block content %}

Sometimes its fine to define an SVG path as

<path d="...">

where the path coordinates are known beforehand and can be supplied.

For example, if we were to define a few paths as follows

<path class="domain" d="M20,20L50,138L170,90L300,240L550,300" style="stroke: red;">
<path class="domain" d="M50,400L125,375L450,225L550,100L350,50L375,175" style="stroke: blue;">
<path class="domain" d="M350,375L50,300L200,50L250,225L450,250" style="stroke: green;">

we would get the following image.


However, if you play with the sliders, you can see what would happen to the visibility of the paths as the size of the SVG element changes. For static pages, changing the size of the element may be unnecessary, but if you want to dynamically resize your page (including the SVG) when a user changes the size of their browser window (or to accommodate smaller screen sizes), then you'll need to scale both the SVG element and its contents.
