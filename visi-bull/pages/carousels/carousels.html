Components of Carousels

Demo placeholder fill patterns to be used in the carousel demos.


Works in three steps: first create carousel container rect, then generate items, then construct carousel around them. Carousel does not care about the items, just needs them to have certain format.

Item construction needs to take:
Number of items to show. Defaults to 1. Heights - an array that if not probided defaults to an array containing one item, which is the height of the carousel containing rect (minus the border width of the items and minus any vertical padding desired). Widths - an array that if nt provided defaults to an array containing one item, which is the width computed in order to make all items fit in the carousel. H padding - an array that if not provided defaults to an array containing only 0. V padding - an array that if not provided default to a pre-specified value (like 10 or something). Items record their own height and width as bound data, along with their horizontal and vertical padding

For carousel need to know:
Whether to center items. Whether to loop infintiely. How many items to skip by each time you advance left or right. Defaults to 1. Can not be larger than the number of items to show.

Infinite Looping (Multiple items - http://kenwheeler.github.io/slick/)

Slide a set number of items out of the viewfinder on arrow click (give some text box where this can be changed).


Centered Carousels

Infinite and non-infinite for all.

Have the center item highlighted in some way, and make it so that the center item is really in the center (require odd number of items). (center mode - http://kenwheeler.github.io/slick/)

Variable width items. The viewport width has to take into account the X widest neighbouring items, where X is the number of items in the viewport at once). (variable width - http://kenwheeler.github.io/slick/)

Click on an item to center it in the carousel.


Unique Carousels

Funky 3Dish carousel with all items in view (http://adam.co/lab/svg/carousel.html).

Autoplaying carousel. Let's you drag and arrow still, but autoplays. Make it break autoplay when you mouse in, and restart when mouse out. (autoplay - http://kenwheeler.github.io/slick/)

One where you can drag the individual items to reorder them and to scroll (also has arrows). i.e. mouse drags the image, mouse ets close to left edge, carousel starts turning left slowly, mouse is released, carousel stops and image drops in

One with a drop down box (or maybe just a context menu on right clicking the carousel) that lets you choose item to jump to, and then rotates carousel to the item.

Fade in and out on transition (best keep it to one item). (fade - http://kenwheeler.github.io/slick/)

Add and remove items (add & remove - http://kenwheeler.github.io/slick/).

Filter items in some way. (filtering - http://kenwheeler.github.io/slick/).

Sync two sliders, one smaller than the other, like a small overview and closeup on focussed on. (Slider syncing - http://kenwheeler.github.io/slick/)
