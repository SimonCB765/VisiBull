Carousel schematic diagram .carousel .itemContainer - dragging operates on this rect.backingRect path.scrollPath g.item .navArrow.left - arrow nav operates on this rect path .navArrow.right - arrow nav operates on this rect path .navDotContainer rect g.navDot - dot nav operates on these rect circle

Parameter options and what they do

items get added data field - transX, transY, distAlongPath, resting
swipe behaviour can be easily added if needed (should work just like dragging)
items passed in will be moved to be under the carousel, not have whatever they currently have as a parent.
If have X items, can only put X - 1 in the items to view
Assumes that the number of items to scroll by is less than the total number of items
can drag, click on arrows/dots to move carousel
clip paths are added (with an id) to eac item to clip them to the carousel
The carousel does not care about the shape of the items, its just places and manipulates them The items can be any shape, provided the height and width given accurately capture the items height and width
navigation arrow can be changed to anything, but need to make sure that the arrow width/height will accomodate the new shape, or clicking will be more fiddly
nav dots can have their shape changed, but need to make sure that the radius provided could encompass the new shape, or clicking will be fiddly
wrapping the items positions around for infinite scrollingis done using d.distAlongPath = (scrollPathLength + d.distAlongPath) % scrollPathLength; this ties the ends of the path together and makes it so that distance 0 === distance scrollPathLength
individual dragging forces it to be single item in view single item scroll by without dots

With centered items it is often best to manually define the carousel size (otherwise can't tell that it is centered).

    ...
