Each item must be the same shape (only needed so tat the clipping works properly when items are being dragged and scrolled in). Here they must all be rectangles, with the width and height of the rects defined as width and height in their data object. The rectangles can be defined as or , either is fine so long as they meet the width and height requirements. It's fairly easy to change the required shape, or to add more in, simply by changing the data object requirements and the clip generating function.

Parameter options and what they do

items get added data field - transX, transY, distAlongPath, resting
swipe behaviour can be easily added if needed (should work just like dragging)
items passed in will be moved to be under the carousel, not have whatever they currently have as a parent.
Need to ensure that carousel is not wide enough to accommodate all items
can drag, click on arrows to move carousel
clip paths are added (with an id) to eac item to clip them to the carousel
The carousel does not care about the shape of the items, its just places and manipulates them The items can be any shape, provided the height and width given accurately capture the items height and width
navigation arrow can be changed to anything, but need to make sure that the arrow width/height will accommodate the new shape, or clicking will be more fiddly
