Usage

Items only need the following entries in their data objects "height": 100, // Height of the item. "key": i, // Unique identifier for the item. "rootID": svgID, // The root of the ID used to refer to the item clip paths. "width": itemWidth // Width of the item.

items get added data field - transX, transY, distAlongPath, resting
swipe behaviour can be easily added if needed (should work just like dragging)
items passed in will be moved to be under the carousel, not have whatever they currently have as a parent.
Need to ensure that carousel is not wide enough to accommodate all items
can drag, click on arrows to move carousel
navigation arrow can be changed to anything, but need to make sure that the arrow width/height will accommodate the new shape, or clicking will be more fiddly
Say that events will trigger on items in carousel for indiv carousel can either create the handlers in such a way that they only fire outside the carousel or can create functions to save ad remove the handlers when the items are added, and then restore them again when the items are moved out of the carousel just need to make sure that there are no handlers that interfere with the dragging of the items themselves
Note that you can have any shaped items for the drag and drop ones
The carousel does not care about the shape of the items, its just places and manipulates them The items can be any shape, provided the height and width given accurately capture the items height and width
